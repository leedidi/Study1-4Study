-- ■■■ 오라클 제거 ■■■

1. 
제어판 > 프로그램 및 기능 → Oracle 관련 항목 제거

(Windows 10 / eXpress Edition 기준)
제어판 > 앱 > 앱 및 기능 → Oracle 관련 항목 제거

2.
실행창 호출(윈도우키 + r) → 『services.msc』 입력

3.
서비스 리스트의 항목을 통해 확인하면
『Orcale』로 시작하는 서비스가 여러 개 확인된다.
즉, Oracle Server 는 서비스를 기반으로 동작한다는 것이다.
위에서 언급한 것 처럼 오라클 프로그램을 제거했다 하더라도
운영체제(OS) 상에서 오라클은 서비스로 동작하기 때무에
이 오라클 서비스를 제거해 주어야 한다.

※ 오라클 서비스를 제거하는 방법

   ·『윈도우키 + r』키 입력(실행창 호출) → 『regedit』입력(레지스트리 편집기 호출)
    //@ 레지스트리 편집기는 절대! 함부로 편집하면 안됨... 레지스트리 편집하면 재부팅시 적용됨....

   ·『HKEY_LOCAL_MACHINE』 > 『SOFTWARE』 > 『ORACLE』 항목 삭제(마우스 우클릭 → 삭제 클릭)

   ·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『CurrentControlSet』 > 『Services』
    → 『Oracle』로 시작하는 모든 항목 삭제(마우스 우클릭 → 삭제 클릭)

   ·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet001』 > 『Services』
    → 『Oracle』로 시작하는 모든 항목 삭제(마우스 우클릭 → 삭제 클릭)

   ·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet002』 > 『Services』
    → 『Oracle』로 시작하는 모든 항목 삭제(마우스 우클릭 → 삭제 클릭)

   ·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet003』 > 『Services』
    → 『Oracle』로 시작하는 모든 항목 삭제(마우스 우클릭 → 삭제 클릭)

   ※ 변경된 레지스트리 정보가 적용되기 위해서는
      반.드.시. 재부팅을 수행해 주어야 한다.

//@ 컴퓨터 사용환경에 따라 ControlSet002,003 없을수도...
//@ 그래도 ControlSet 붙은 애들 안에 있는 Orcale 붙은 항목들은 삭제해 주어야 함!


4. 
재부팅 이후 윈도우 탐색기에서 오라클 홈과 관련된 모든 내용을
물리적으로 삭제한다.


5.
또한 데이터 파일 경로 및 설치 경로의 모든 디렉터리와 파일들을
물리적으로 삭제할 수 있도록 한다.
